TABLE: jobs (主工作表)
================================================================================
主要工作原始信息记录表，存储所有制造作业的信息
  - oe_number - 订单录入号
  - job_number - 作业号
  - customer_name - 客户名称
  - job_quantity - 作业数量
  - drawing_number - 零件号
  - revision - 修订版本
  - customer_contact - 客户联系方式
  - drawing_release - 图纸发布
  - line_number - 订单录入行号
  - part_description - 零件描述
  - unit_price - 单价
  - po_number - 采购订单号
  - packing_slip - 装箱单
  - packing_quantity - 装箱数量
  - invoice_number - 发票号
  - delivery_required_date - 所需交货日期 (YYYY-MM-DD)
  - delivery_shipped_date - 发货日期 (YYYY-MM-DD)
  - create_timestamp - 创建时间戳
  - last_modified - 最后修改时间
  - unique_key UNIQUE - 唯一键 (job_number|line_number)
  - priority DEFAULT 'Normal' - 优先级（Critical|Urgent|Important|Normal|Minor|Hold）

3NF
customer
存储客户及其联系人的基本信息。将 Contact 归于客户或任务均可，但通常联系人属于客户方。
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK)
customer_name: TEXT UNIQUE NOT NULL
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

customer_contact
为了实现一个客户对应多个联系人，我们需要将联系人从客户表中剥离出来，建立独立的联系人表。
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK)
customer_id: INTEGER NOT NULL (FK -> customer.id) - 归属哪个客户
contact_name: TEXT NOT NULL DEFAULT '' - 联系人姓名
contact_email: TEXT - 邮箱
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

purchase_order
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK)
po_number: TEXT UNIQUE KEY NOT NULL DEFAULT ''
oe_number: TEXT NOT NULL DEFAULT ''
contact_id: INTEGER NOT NULL (FK -> customer_contact.id) - 直接关联到具体的某位联系人
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

job
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK)
job_number: TEXT UNIQUE KEY NOT NULL
po_id: INTEGER (FK -> purchase_order.id)
priority: TEXT NOT NULL DEFAULT "normal"
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

order_item
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK)
job_id: INTEGER NOT NULL (FK -> production_job.id)
part_id: INTEGER NOT NULL (FK -> part.id)
line_number: INTEGER NOT NULL
order_quantity: INTEGER NOT NULL
actual_price: REAL - 签约时的合同价
drawing_release_date: TEXT NOT NULL - ISO8601 字符串
delivery_required_date: TEXT NOT NULL - ISO8601 字符串
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

shipment
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK) - 自增主键
packing_slip_number: TEXT UNIQUE KEY NOT NULL DEFAULT '' - 装箱单号
invoice_number: TEXT DEFAULT '' - 发票号
delivery_shipped_date: TEXT NOT NULL DEFAULT '' - 实际发货日期
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

shipment_item
为了消除line与shipment之间的多对多的关系所引入的中间表
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK) - 自增主键
order_item_id: INTEGER NOT NULL (FK -> order_item.id) - 关联到具体的一行
shipment_id: INTEGER NOT NULL (FK -> shipment.id) - 关联到一次发货
shipped_quantity: INTEGER NOT NULL - 本次发货对于某个零件的发货数量
note: TEXT NOT NULL DEFAULT '' - 针对本次发货的说明
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

part
零件的描述和价格在 CSV 中随零件号重复，应独立建表以消除冗余。
id: INTEGER PRIMARY KEY AUTOINCREMENT (PK)
drawing_number: TEXT NOT NULL UNIQUE
revision: TEXT NOT NULL
description: TEXT NOT NULL DEFAULT ''
unit_price: REAL
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串

note
id:	INTEGER	PK AUTOINCREMENT
po_id: INTEGER	FK -> purchase_order.id -	可为空 (NULL)
job_id:	INTEGER	FK -> job.id - 可为空 (NULL)
order_item_id: INTEGER	FK -> order_item.id	- 可为空 (NULL)
part_id: INTEGER	FK -> part.id	- 可为空 (NULL)
content: TEXT	NOT NULL DEFAULT ''	- 备注内容
created_at:	DATETIME DEFAULT CURRENT_TIMESTAMP	- 记录时间
updated_at: TEXT NOT NULL - ISO8601 字符串

drawing_file
该表用于记录从 G 盘扫描到的所有 PDF 文件信息。通过 part_id 实现与零件表的关联。对于 1.pdf 等无法自动识别的文件，预留 is_manual_confirmed 字段用于人工标记。
id: INTEGER PRIMARY KEY AUTOINCREMENT
part_id: INTEGER (FK -> part.id) - 关联到的具体零件，允许为 NULL（代表扫描到但尚未匹配成功的图纸）
file_name: TEXT NOT NULL - 原始文件名（例如：123-456_RevA_REDLINE.pdf）
file_path: TEXT UNIQUE NOT NULL - 文件在 G 盘的全路径（作为唯一约束，防止重复记录）
parent_folder: TEXT - 父文件夹名称（对于像 1.pdf 这种文件，文件夹名往往是唯一的线索）
is_active: INTEGER DEFAULT 1 - 逻辑标记：1 代表该零件当前推荐查看的版本，0 代表历史版本
is_manual_confirmed: INTEGER DEFAULT 0 - 人工确认标记：1 代表该关联由人工核对确认，防止脚本扫描覆盖
last_modified_at: TEXT - 文件的最后修改时间（用于判断哪个文件是最新的）
created_at: TEXT NOT NULL - ISO8601 字符串
updated_at: TEXT NOT NULL - ISO8601 字符串
